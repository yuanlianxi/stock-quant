## 3. 推荐首选：VN.py

### 3.1 为什么推荐 VN.py

| 优势 | 说明 |
|------|------|
| **CTP接口** | 直连国内149家期货公司，支持全部商品期货 |
| **生态完整** | 回测、实盘、风控、套利、期权全支持 |
| **社区活跃** | 文档完善，社区讨论活跃 |
| **开源免费** | GitHub开源（vnpy/vnpy）|
| **对接方便** | 可与我们Phase 2的回测系统对接 |

### 3.2 VN.py 架构

```
CTP接口（期货公司）
    ↓
VN.py Core（核心引擎）
    ↓
┌─────────────────────────────────────┐
│  策略引擎  │  风控模块  │  套利模块  │  算法交易  │
└─────────────────────────────────────┘
    ↓
下单执行（订单路由）
```

### 3.3 核心功能

| 模块 | 功能 |
|------|------|
| **CTA策略** | 趋势跟踪、反转、套利策略 |
| **回测引擎** | 历史数据回测、参数优化 |
| **风控模块** | 仓位管理、止损、风险监控 |
| **算法交易** | VWAP、TWAP、冰山订单 |
| **数据服务** | Tick级数据、实时行情 |
| **Exchange** | 直连期货公司CTP |

## 4. 对比我们自建回测系统

### 4.1 定位对比

| 维度 | VN.py | 我们自建系统 |
|------|-------|-------------|
| **阶段** | 实盘对接 | 回测验证 |
| **接口** | CTP直连 | 数据获取层 |
| **策略** | 直接执行 | 策略开发测试 |
| **费用** | 免费（CTP手续费自付）| 免费开源 |

### 4.2 推荐路径

```
Phase 2-A：回测系统（Backtrader）
    ↓ 策略验证有效
Phase 2-C：VN.py 实盘对接
```

**为什么这样设计：**
1. Backtrader 回测验证策略有效性（低成本、可视化）
2. 策略稳定后，再花时间对接 VN.py 实盘
3. 避免直接在实盘测试亏损策略

## 5. 其他软件简要说明

### 5.1 EasyTrader

- **定位**：聚宽出品，对接聚宽实盘
- **优点**：与聚宽研究平台无缝衔接
- **缺点**：需要聚宽账号，平台依赖
- **适合**：已在聚宽研究的用户

### 5.2 XTP

- **定位**：东方证券量化平台
- **优点**：东方证券官方支持
- **缺点**：只能对接东方证券
- **适合**：使用东方证券的用户

### 5.3 Backtrader

- **定位**：我们已在用的回测框架
- **优点**：轻量级、事件驱动、文档完善
- **缺点**：不直接支持实盘
- **适合**：策略回测验证
