## 1. 交易系统概述

T1基金交易系统V1.0 是一套基于海龟交易法则改编的**趋势跟踪商品期货系统**，于2019年9月正式形成文档版本。该系统在原版海龟的基础上针对中国境内大宗商品期货市场做了参数调整和风控增强，适用于黄金、白银、螺纹、铁矿等国内主流期货品种。

**系统定位**：
- 纯多空趋势跟踪，不做选股
- 日内突破建仓，不等收盘确认
- ATR波动率自适应仓位
- 最大4Unit持仓，全程硬止损

**核心特点**：
- 入市周期拉长至55日（相比原版20日），过滤更多假突破
- 出场周期20日反突破（相比原版10日），让利润更充分奔跑
- 内置基金风控框架（清盘线、名义总资金动态调整）
- 明确的交割月前退出机制

## 2. 入市规则（55日突破）

### 2.1 入市条件

**做多入场**：当前价格 > 前55日最高价 → 在突破方向建立头寸

**做空入场**：当前价格 < 前55日最低价 → 在突破方向建立头寸

### 2.2 执行要点

- **日间突破交易**：不等收盘或次日开盘，盘中突破即刻建仓
- **跳空突破**：若开盘已跳空越过55日高低点，则以开盘价直接建仓
- 不需要收盘价确认信号

### 2.3 与原版海龟对比

| 项目 | T1系统 | 原版海龟 |
|------|--------|---------|
| 入市周期 | 55日 | 20日（经典）/ 28日（原版） |
| 信号确认 | 日内突破，不等收盘 | 通常等收盘确认 |
| 跳空处理 | 开盘即建仓 | 视情况而定 |

**设计意图**：55日周期显著长于原版，筛选门槛更高，入场信号更少但质量更高，更适合期货市场的低信噪比环境。

## 3. 加仓规则（N/ATR系统）

### 3.1 N值（ATR）计算

```
N = ATR（20日指数移动平均）
N = (19 × PDN + TR) / 20
```

其中：
- **PDN**：前一日N值
- **TR**：真实波幅 = max(今日最高-今日最低, 今日最高-昨日收盘, 今日最低-昨日收盘)

### 3.2 头寸规模单位

```
1个Unit = (总资金 × 1%) / (N × 每点人民币)
```

含义：总资金的1%作为单笔交易的风险上限，除以N值（波动率）得到实际可交易的手数。

### 3.3 加仓节奏

| 次数 | 建仓价格 | 说明 |
|------|---------|------|
| 第1次 | 突破入场价 | 入市即建1个Unit |
| 第2次 | 入场价 + 0.5N | 价格每涨0.5N增加1个Unit |
| 第3次 | 入场价 + 1.0N | 同上 |
| 第4次 | 入场价 + 1.5N | 最多4个Unit |

### 3.4 跳空加仓处理

若跳空无法在设定点位成交（跳过0.5N间隔），则以**开盘价或盘中市价**直接追入。

### 3.5 与原版海龟对比

| 项目 | T1系统 | 原版海龟 |
|------|--------|---------|
| ATR周期 | 20日EMA | 通常14日 |
| 加仓间隔 | 0.5N | 0.5N（一致） |
| 最大持仓 | 4个Unit | 4个Unit（一致） |
| 跳空加仓 | 允许以市价追入 | 无明确说明 |

## 4. 止损规则

### 4.1 止损体系

| 维度 | 规则 |
|------|------|
| 单笔交易风险上限 | 总资金的 **2%** |
| 最大止损幅度 | 价格波动 **2N** |
| 加仓时止损动态调整 | 每次加仓后，前面所有单位的止损价格上移 **1/2N** |
| 跳空情况 | 前面仓位止损**不动态调整**（保持原止损位）|

### 4.2 止损逻辑图

```
入场建仓1个Unit
    ↓ 价格下跌2N
止损全部平仓（亏损2%）
    
或者：
入场建仓1个Unit
    ↓ 价格每上涨0.5N
加仓1个Unit，同时前面止损上移1/2N
    ↓ 最多4个Unit
    ↓ 任一时刻跌破动态止损线
全部平仓
```

### 4.3 止损动态调整示例

假设入场价100元，N=2元：
- 第1Unit止损：100 - 2×2 = 96元
- 价格涨到102元（+1N），加仓第2Unit，止损上移：96 + 1 = 97元（前1个Unit止损）
- 价格再涨到104元（+2N），加仓第3Unit，前2个Unit止损再上移1元：97+1=98元，98+1=99元
- 跳空时前面仓位止损不调整

### 4.4 与原版海龟对比

| 项目 | T1系统 | 原版海龟 |
|------|--------|---------|
| 单笔风险上限 | 2% | 2%（一致） |
| 最大止损 | 2N | 2N（一致） |
| 加仓止损上移 | 1/2N | 无明确动态调整规则 |
| 跳空止损处理 | 跳空时前面仓位止损不调整 | 无明确说明 |

## 5. 退出规则（20日反突破）

### 5.1 出场条件

| 方向 | 触发条件 |
|------|---------|
| 多头出场 | 价格 < 过去20日最低价 |
| 空头出场 | 价格 > 过去20日最高价 |

### 5.2 出场与止损的关系

- **止损**：风险控制，触发后全部平仓
- **出场**：趋势结束信号，触发后全部平仓
- 两者同时生效，取先触发者

### 5.3 与原版海龟对比

| 项目 | T1系统 | 原版海龟 |
|------|--------|---------|
| 出场周期 | 20日 | 10日（经典）/ 20日（原版） |
| 出场逻辑 | 反突破 | 反突破（一致） |

**设计意图**：T1使用20日出场与原版丹尼斯版本一致，目的是让利润充分奔跑，在趋势行情中持有更长时间。

## 6. 交易品种

### 6.1 交易品种清单

T1系统覆盖国内商品期货市场共 **35个品种**，按类别划分：

**金属类**：
- 贵金属：黄金、白银
- 黑色系：铁矿、螺纹、焦炭、热卷、焦煤
- 有色金属：铜、铝、锌、镍、纸浆

**能源化工类**：
- 原油、燃油、沥青、PTA、郑醇、PP（聚丙烯）、塑料、PVC、乙二醇

**农产品类**：
- 软商品：白糖、棉花
- 油脂油料：豆粕、豆油、菜粕、郑油、棕榈
- 饲料：玉米
- 特色品种：橡胶、鸡蛋、苹果

### 6.2 强关联品种提示

| 关联组 | 品种 |
|--------|------|
| 贵金属组 | 黄金 ↔ 白银 |
| 黑色系组 | 铁矿 ↔ 螺纹 |
| 油脂油料组 | 豆粕 ↔ 豆油 ↔ 菜粕 ↔ 郑油 |

**注意**：强关联品种同时持仓可能造成风险集中，应关注总体仓位均衡。

### 6.3 交割月规则

- **任何头寸不得进入交割月**
- **强制退出时间**：交割月前最后交易日 **14:30**

### 6.4 与原版海龟对比

| 项目 | T1系统 | 原版海龟 |
|------|--------|---------|
| 市场 | 国内商品期货 | 美国期货（含外汇、债券） |
| 品种数量 | 35个 | 无限制（美国市场） |
| 关联品种处理 | 明确提示 | 无明确说明 |
| 交割月规则 | 有强制退出机制 | 无此问题（美国期货规则不同）|

## 7. 风控方案

### 7.1 基金规模与资金结构

| 项目 | 详情 |
|------|------|
| 总规模 | 200万元 |
| 出资方 | l、oy各50%（各100万） |
| 出资节奏 | 分两期：第一期100万（2019.11.14截止）|

### 7.2 入市时间线

| 时间 | 名义总资金 |
|------|-----------|
| 2019.11.18 - 2019.11.31 | 50万 |
| 2019.12.1起 | 经委员会同意可上调至100万 |

### 7.3 名义总资金动态调整规则

| 月末净值状态 | 名义总资金调整 |
|-------------|--------------|
| 内在价值 > 名义总资金 | 上调至内在价值（IV） |
| 净值 < 初始90% | 调为初始的 **0.8倍** |
| 净值 < 初始80% | 静默期1个月，调为初始的 **0.64倍** |

> **内在价值（IV）**：通常指基金当前的实际净资产价值。

### 7.4 强制清盘线

**净值 < 初始70% → 强制清盘**

### 7.5 风险度控制

| 条件 | 操作 |
|------|------|
| 风险度 > 90% | 必须减仓 |
| 砍仓优先级 | 亏损金额最高的合约优先平仓 |

### 7.6 与原版海龟风控对比

| 项目 | T1系统 | 原版海龟 |
|------|--------|---------|
| 风控框架 | 基金级完整方案 | 无（纯策略风控）|
| 名义总资金 | 动态调整（3档） | 无此概念 |
| 清盘线 | 70% | 无此概念 |
| 风险度上限 | 90%必须减仓 | 无此量化指标 |
| 砍仓优先级 | 亏损最多优先 | 无明确规定 |
| 静默期 | 净值<80%时静默1个月 | 无 |

**T1风控特点**：T1风控方案远超原版海龟策略本身，是一套完整的基金级风控体系，包含了资金管理（名义总资金动态调整）、清盘线、风险度监控和强制减仓机制。

## 8. 与原版海龟对比

### 8.1 核心参数对比总表

| 维度 | T1系统 | 原版海龟（丹尼斯） | 原版海龟（经典20/10） |
|------|--------|------------------|-------------------|
| 入市周期 | **55日** | 28日 | 20日 |
| ATR周期 | **20日EMA** | 20日 | 14日 |
| 出场周期 | **20日** | 20日 | 10日 |
| 加仓间隔 | 0.5N | 0.5N | 0.5N |
| 最大持仓 | 4个Unit | 4个Unit | 4个Unit |
| 单笔风险 | 2% | 2% | 2% |
| 最大止损 | 2N | 2N | 2N |
| 日内突破处理 | **直接建仓不等收盘** | 等收盘 | 等收盘 |
| 跳空加仓 | **允许市价追入** | 无明确规定 | 无明确规定 |
| 止损动态调整 | **加仓时前面仓位止损上移1/2N** | 无明确规则 | 无明确规则 |
| 跳空止损处理 | **前面仓位止损不调整** | 无 | 无 |

### 8.2 关键差异分析

**① 入市周期55日 vs 20日**
T1使用55日突破是极其保守的设置，在原版海龟中属于"长期系统"。这意味着：
- 信号数量大幅减少（仅 Captures 更大级别的趋势）
- 假突破过滤效果显著增强
- 趋势跟踪周期更长，利润空间更大
- 但可能错过中等规模趋势，机会成本增加

**② 日内突破直接建仓**
原版海龟通常要求等收盘确认信号以避免日内假突破。T1选择直接建仓：
- 优点：不等待收盘，避免跳空导致无法入场
- 缺点：日内假突破会被捕捉，增加交易成本
- **本质上是牺牲部分信号质量换取更好的入场价位**

**③ 止损动态上移机制**
T1在每次加仓后，将前面所有Unit的止损价格统一上移1/2N（而非仅调整最新加仓的Unit止损），这是比原版更激进的跟踪止损机制：
- 持仓越重，整体止损线越高
- 有利于锁定更多利润
- 但如果行情震荡，加仓后止损上移可能触发后续止损

**④ 基金级风控 vs 纯策略风控**
T1的风控方案远超原版海龟策略本身，是一套完整的私募基金风控体系：
- 资金端：名义总资金动态调整，避免过度暴露
- 运营端：清盘线、风险度监控、静默期
- 交易端：砍仓优先级、交割月退出机制
- 这套风控体系与策略本身同等重要

### 8.3 T1系统适应场景分析

| 市场环境 | T1表现预期 | 原因 |
|---------|----------|------|
| 大趋势行情（持续数月） | ⭐⭐⭐⭐⭐ | 55日捕捉大趋势，20日跑出利润 |
| 中级趋势（1-3月） | ⭐⭐⭐ | 55日可能过滤部分中级趋势 |
| 震荡行情 | ⭐⭐ | 多次假突破被止损，但好于短线系统 |
| 跳空频繁的品种 | ⭐⭐⭐ | 日内建仓+跳空市价追入设计能捕捉跳空 |
| 高波动品种（原油、铁矿）| ⭐⭐⭐⭐ | ATR自适应仓位有效控制风险 |

## 9. 要点总结

### 9.1 T1系统核心框架

```
【入市】55日突破 → 日内直接建仓（不等收盘）
    ↓
【加仓】每+0.5N加1个Unit，最多4个Unit
    ↓
【止损】2N止损；加仓时前面Unit止损上移1/2N（跳空除外）
    ↓
【出场】20日反突破（跌破20日最低/涨破20日最高）
    ↓
【风控】基金级：清盘线70%、风险度90%强制减仓、交割月前退出
```

### 9.2 三大创新点

**① 55日超长期入市**：相比经典20日，极大过滤假信号，专做大趋势

**② 止损动态上移**：加仓即锁定已有利润，整体风险随持仓增加而收紧

**③ 基金级风控体系**：名义总资金动态调整 + 清盘线 + 静默期，超出纯策略范畴

### 9.3 实施注意事项

- 35个期货品种需按合约乘数分别计算Unit，不能用统一公式
- 强关联品种（黄金-白银、铁矿-螺纹等）需控制总体仓位不超过20%账户风险
- 交割月前14:30强制平仓，需在交易系统中预设预警和自动执行
- 基金净值每月评估一次，名义总资金调整后下一月生效
- 跳空市价追入需评估滑点风险，建议实盘设置限价单追入

### 9.4 与后续策略开发关联

T1系统可作为**stock-quant项目**的趋势跟踪策略基准，与0005号海龟调研文档结合使用：
- 回测对比：55日 vs 20日 vs 28日不同入市周期
- 参数优化：加入自适应出场、动态止损等改良
- 多策略组合：趋势跟踪（T1）+ 均值回归（布林带）双系统
