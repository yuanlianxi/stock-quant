## 2. CTP 接口配置详解

### 2.1 什么是 CTP

CTP（Comprehensive Transaction Platform）是上海期货交易所推出的 **T+0 衍生品交易系统**，是国内期货实盘的主流接口。

| 项目 | 内容 |
|------|------|
| 全称 | Future Secutities Transaction Platform |
| 运营商 | 上海期货交易所信息技术 |
| 特点 | 穿透式监管、T+0、保证金 |
| 接口类型 | 期货交易接口（CTP、华泰、主席）|

> VN.py 支持多种交易接口：CTP（金主要）、CTP证券、期货账户证券户（继承CTP）、XTP（股票）等

### 2.2 CTP 账号类型

| 类型 | 说明 | 用途 |
|------|------|------|
| **SimNow** | 模拟盘 | 策略验证、测试 |
| 期货公司 simnow | 149家期货公司模拟 | 验证账户 |
| 实盘账户 | 正式交易 | 真实资金 |

### 2.3 CTP 配置文件内容

VN.py 连接 CTP 需要以下配置信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `user_name` | 账号 | `your_account` |
| `password` | 密码 | `your_password` |
| `broker_id` | 经纪商代码 | `9999`（SimNow）|
| `trade_front_addr` | 交易服务器地址 | `tcp://218.202.42.xxx:xxxxx` |
| `market_front_addr` | 行情服务器地址 | `tcp://218.202.42.xxx:xxxxx` |
| `auth_code` | 认证码 | SimNow 模拟可留空 |
| `product_info` | 产品信息 | 可自定义 |

### 2.4 SimNow 配置信息

**SimNow** 是期货公司提供的官方模拟盘，连接信息：

| 项目 | 内容 |
|------|------|
| Broker ID | `9999` |
| 交易服务器 | `tcp://218.202.42.226:10102` |
| 行情服务器 | `tcp://218.202.42.226:10103` |
| 注册地址 | https://www.simnow.com.cn |

> 注意：SimNow 地址会变化，建议使用时从官网重新确认

### 2.5 SimNow 注册流程

1. 访问 https://www.simnow.com.cn
2. 点击注册，填写手机号、密码
3. 获取 6 位数的 `investor_id`（模拟账号）
4. 记录 `password`（模拟密码）
5. 绑定期货公司（可选 149 家）

## 3. VN.py CtaStrategy 策略模板解析

### 3.1 CtaStrategy 模块位置

```

│   └── cta_strategy.py    # 策略基类模板
├── strategies/

│   ├── r_breakout.py      # R-Breaker 策略示例
│   └── ...                # 其他示例策略
└── engine.py              # CTA 引擎
```

### 3.2 策略模板核心结构

```python
from vnpy_ctastrategy import CtaStrategy

class MyStrategy(CtaStrategy):
    """自定义 CTA 策略"""

    # 策略参数（暴露给 UI 可调整）
    fast_ma: int = 10
    slow_ma: int = 20
    parameters = ["fast_ma", "slow_ma"]

    # 策略变量（记录内部状态）
    pos: int = 0  # 持仓量
    fast_ma_value: float = 0
    slow_ma_value: float = 0
    variables = ["pos", "fast_ma_value", "slow_ma_value"]

    def __init__(self, cta_engine, strategy_name, vt_symbol, setting):
        """"""
        super().__init__(cta_engine, strategy_name, vt_symbol, setting)
        # 使用 K 线时间序列管理模块（用于技术指标计算）
        self.am = ArrayManager()

    def on_init(self):
        """策略初始化（加载历史数据）"""
        self.write_log("策略初始化")
        self.load_bar(10)  # 加载 10 天历史 K 线

    def on_start(self):
        """策略启动"""
        self.write_log("策略启动")

    def on_stop(self):
        """策略停止"""
        self.write_log("策略停止")

    def on_bar(self, bar: BarData):
        """K 线推送（核心交易逻辑）"""
        # 更新 K 线序列
        self.am.update_bar(bar)
        if not self.am.inited:
            return

        # 计算技术指标
        self.fast_ma_value = self.am.ma(self.fast_ma)
        self.slow_ma_value = self.am.ma(self.slow_ma)

        # 交易逻辑
        if self.pos == 0:  # 无持仓
            if self.fast_ma_value > self.slow_ma_value:
                self.buy(bar.close, 1)  # 买入开仓
        elif self.pos > 0:  # 持多仓
            if self.fast_ma_value < self.slow_ma_value:
                self.sell(bar.close, 1)  # 卖出平仓

    def on_trade(self, trade: TradeData):
        """成交回报推送"""
        self.put_event()

    def on_order(self, order: OrderData):
        """委托回报推送"""
        pass

    def on_calculate(self):
        """定量计算（用于回测/复盘）"""
        pass
```

### 3.3 ArrayManager 使用详解

`ArrayManager` 是 VN.py 内置的 K 线序列管理模块，用于高效计算技术指标：

| 方法 | 说明 |
|------|------|
| `update_bar(bar)` | 更新单根 K 线 |
| `inited` | 是否已初始化完成（需要至少 100 条数据）|
| `ma(period)` | 计算移动平均线 |
| `atr(period)` | 计算 ATR（真实波幅）|
| `atr_percentile(period)` | ATR 百分位 |

```python
# 示例：初始化 + 计算双均线
self.am = ArrayManager()

def on_bar(self, bar: BarData):
    self.am.update_bar(bar)
    if not self.am.inited:
        return

    # 移动平均线
    ma5 = self.am.ma(5)
    ma20 = self.am.ma(20)

    # ATR（用于止损）
    atr10 = self.am.atr(10)
```

## 4. VN.py 实盘运行流程

### 4.1 启动脚本模板（run_cta.py）

```python
from vnpy.event import EventEngine
from vnpy.trader.engine import MainEngine
from vnpy.trader.ui import MainWindow, create_qapp
from vnpy_ctp import CtpGateway
from vnpy_ctastrategy import CtaStrategyApp
from vnpy_ctabacktester import CtaBacktesterApp
from vnpy_datamanager import DataManagerApp


def main():
    """启动 VN.py Trader"""
    qapp = create_qapp("vnpy_trader")

    event_engine = EventEngine()
    main_engine = MainEngine(event_engine)

    # 添加交易接口
    main_engine.add_gateway(CtpGateway)

    # 添加应用模块
    main_engine.add_app(CtaStrategyApp)
    main_engine.add_app(CtaBacktesterApp)
    main_engine.add_app(DataManagerApp)

    main_window = MainWindow(main_engine, event_engine)
    main_window.showMaximized()

    qapp.exec()


if __name__ == "__main__":
    main()
```

### 4.2 连接 CTP 接口

启动 run_cta.py 后：

1. 在 VN Trader 主界面，点击【系统】→【连接交易接口】
2. 在弹出的对话框中填写 CTP 配置（用户名、密码、经纪商、地址）
3. 点击【连接】
4. 等待日志输出「合约信息查询成功」

### 4.3 添加并运行策略

1. 左侧菜单点击【功能】→【CTA 策略】
2. 左上角下拉框选择策略类名（不是文件名）
3. 点击【添加策略】：
   - 实例名称（如 `dual_ma_rb2505`）
   - 合约品种（如 `rb2505.SHFE`）
   - 策略参数（参数值）
4. 点击【初始化】，等待【inited】变为 True
5. 点击【启动】，等待【trading】变为 True

### 4.4 停止策略

点击策略实例下的【停止】按钮，CTA 引擎自动撤销所有活动委托，保存状态到 `cta_strategy_data.json`。

## 5. 数据配置（DataFeed）

### 5.1 数据源类型

| 数据源 | 说明 | 获取方式 |
|--------|------|---------|
| **RQData** | 米筐数据服务 | 需要单独订阅 |
| **XTP** | 迅投数据服务 | 需要 XTP 账户 |
| **Tushare** | 开源数据 | 需要 token |
| **本地数据库** | SQLite | 通过 DataRecorder 录制 |

### 5.2 RQData 配置

```python
# 在启动脚本中添加
from vnpy_rqdata import RqdataDataService

# 初始化
rqdata = RqdataDataService()
rqdata.init_query()  # 需要在主引擎启动后调用
```

然后在 VN Trader 全局配置中填写 RQData 的用户名密码。

### 5.3 数据录制（DataRecorder）

使用 DataRecorder 录制实时行情到本地数据库：

1. 在 VN Trader 中添加 DataRecorder 应用
2. 订阅需要录制的合约
3. 数据自动存入 SQLite 数据库

## 6. 多策略 + 多账号管理

### 6.1 多策略

VN.py CTA 策略支持**同一个策略类多个实例**：

- 实例 A：`dual_ma_rb2505`（交易螺纹钢 rb2505）
- 实例 B：`dual_ma_hc2505`（交易热卷 hc2505）

每个实例参数独立，互不影响。

### 6.2 多账号

VN.py 支持同时连接多个 CTP 账户（或不同接口）：

```python
# 添加多个 CTP 网关
main_engine.add_gateway(CtpGateway)  # 账户 A
main_engine.add_gateway(CtpGateway)  # 账户 B（不同配置）
```

策略实例绑定 vt_symbol，可绑定到不同网关的账户。

### 6.3 策略隔离

- 每个策略实例独立运行
- 持仓、参数、变量互不干扰
- 策略崩溃不影响其他策略

## 7. Mac 系统 VN.py 安装注意事项

### 7.1 Mac 安装要点（来自官方文档）

1. Python 3.10（推荐）
2. TA-Lib C++ 开发包：`brew install ta-lib`
3. NumPy + TA-Lib Python：`pip install ta-lib`
4. VN.py 核心框架 + 插件模块
5. CTP 接口需从 GitHub 克隆源码本地编译安装

### 7.2 Mac CTP 动态库信任

编译完成后，需要手动将两个动态库添加到系统信任：

```
~/github/vnpy_ctp/api/libs/thostmduserapi_se.framework/Versions/A/thostmduserapi_se
~/github/vnpy_ctp/api/libs/thosttraderapi_se.framework/Versions/A/thosttraderapi_se
```

操作方式：在【访达】中找到文件 → 双击打开 → 系统提示「已阻止」→【系统偏好设置】→【安全性与隐私】→【仍要打开】

### 7.3 Mac 地址配置注意事项

由于 Mac CTP API 结构变化（6.7.2+），编译安装时需要：
- 源码目录不能删除或移动（动态库路径硬编码）
- 建议放在固定位置如 `~/github/vnpy_ctp/`

## 8. 实盘账户开户指引

### 8.1 开户流程

1. 选择期货公司（如 `南华期货`、`永安期货`、`中信期货`）
2. 线上/线下提交资料（身份证、银行卡）
3. 金融适当性评估（需通过）
4. 期货账户开通（商品期货 + 金融期货）
5. 下载交易软件（快期、易盛等）
6. 获得 CTP 相关账号（`investor_id`、`password`、`broker_id`）

### 8.2 注意事项

- **保证金**：每家期货公司保证金比例略有差异
- **CTP 主席 vs 分主席**：主席速度更快，但分主席可多账户
- **穿仓风险**：CTA 策略需设置硬止损（2N），防止穿仓
